## 1. Введение
Гибридные модели программирования, сочетающие MPI (Message Passing Interface) и OpenMP, позволяют эффективно использовать вычислительные ресурсы современных кластерных систем. В такой схеме MPI отвечает за распределение данных между узлами (процессами), а OpenMP — за параллелизм внутри каждого узла на уровне ядер процессора. Поразрядная сортировка (Radix Sort), обладая линейной сложностью, хорошо поддается такому двухуровневому распараллеливанию.

## 2. Постановка задачи
Реализовать гибридный алгоритм поразрядной сортировки для 32-битных целых чисел.
- **Входные данные**: Вектор `std::vector<int32_t>`.
- **Выходные данные**: Отсортированный вектор на нулевом процессе.
- **Особенности**: Использование MPI для передачи сообщений и OpenMP для ускорения локальных вычислений в каждом процессе.

## 3. Описание алгоритма (последовательный)
Последовательная реализация основана на алгоритме LSD (Least Significant Digit):
1. Определение диапазона значений для учета отрицательных чисел.
2. Поразрядный проход (обычно по 10-ичному основанию или по байтам) с использованием сортировки подсчетом.
3. Обеспечение стабильности сортировки на каждом этапе.

## 4. Схема распараллеливания
Реализована гибридная схема "MPI + OpenMP":
1. **Распределение (MPI)**: Корневой процесс (rank 0) делит входной массив на части и рассылает их остальным процессам с помощью `MPI_Scatterv`.
2. **Локальная параллельная сортировка (OpenMP)**: Внутри каждого MPI-процесса полученный блок данных сортируется методом `LocalRadixSort`, который распараллелен с помощью OpenMP-директив (обработка разрядов и слияние подблоков).
3. **Сбор данных (MPI)**: Отсортированные блоки собираются на корневом процессе (`MPI_Gatherv`).
4. **Финальное слияние**: Корневой процесс выполняет итоговое слияние (merge) отсортированных частей в единый массив.

## 5. Детали реализации
- **Класс**: `VotincevDRadixMergeSortALL`.
- **MPI-функции**: `MPI_Bcast` для передачи размера, `MPI_Scatterv`/`MPI_Gatherv` для работы с блоками разного размера (если данные не делятся нацело).
- **OpenMP-ускорение**: Внутри `LocalRadixSort` используются `#pragma omp parallel` для разделения локального блока между потоками.
- **Память**: Каждый процесс выделяет память только под свою часть данных и временные буферы, что оптимизирует использование ОЗУ на распределенных системах.

## 8. Заключение
Гибридная реализация MPI+OpenMP продемонстрировала эффективность, близкую к чистому OpenMP на одном узле. Основное преимущество технологии ALL проявляется при масштабировании на несколько физических узлов кластера, где MPI обеспечивает межсетевое взаимодействие, а OpenMP — эффективную работу с общей памятью внутри узла.
